# §C2. Второй заход в конфигуратор — РБ и РР

<!-- TODO: написать. Объём 3000–4500 слов (самый длинный после B3.5). -->

**Что в этом параграфе:**

**Регистр бухгалтерии (РБ):**
- Ярлыки UI читаются (после C1).
- **Образ:** сеть переводов («−тут +там»), каждый перевод сходится.
  **Не** «сообщающиеся сосуды».
- **Неубиваемая ценность РБ:** платформа сама следит за балансом двух сторон каждой операции.
- Корреспонденция опциональна; без неё РБ ≈ «вид РН с признаком активности».
- Субконто — «у места своя разбивка».
- Запись выглядит иначе (проводка с парой счетов, не просто число) — это якорь **механики**,
  не предметки.

**Регистр расчёта (РР):**
- Полная отсылка к врезке §5 PLAN (которая раскрыта ещё в A4).
- **Образ для айтишника:** «вытесняющая многозадачность / календарь с правилами пересечения»
  (preemption — то же слово).
- Раскрытие лейбла «копилка с машиной времени» (из A4).
- Виртуалки: `База<Имя>`, `ДанныеГрафика`, `ФактическийПериодДействия`.
  **Имена сверить с синтакс-помощником целевой версии перед релизом** (CLAUDE §11).

**Канон (нельзя):** «сообщающиеся сосуды» для РБ; «РР = накопление с допсвойствами».

**Мостик к C3:** «Все четыре характера в руках. Теперь — дерево выбора. C3».
