# Конституция книги

> Контракт с читателем и ограничитель для автора.
> Claude Code читает этот файл перед каждым действием.

---

## Что эта книга ЕСТЬ

- Учебная книга о регистрах «1С:Предприятие 8», ведущая читателя от общего скелета
  (измерения/ресурсы/реквизиты) до **осознанного выбора** регистра.
- Часть серии «1С как иностранный язык»: продолжает супра-метафору
  существительное→справочник, прилагательное→реквизит, глагол→документ, регистр→книга учёта.
- Опора на **родной домен айтишника** (логи, расписание, кошелёк, история операций по карте).
- Три уровня подачи: 0 — бытовой вход; 1 — основной курс; 2 — экспертный конспект-награда.
- Книга платформы, не предметки. Бухгалтерская/расчётная терминология вводится точечно
  и только там, где без неё нельзя (фаза C).

## Что эта книга НЕ ЕСТЬ

- Не учебник бухучёта, зарплаты или регламентированного учёта.
- Не справочник по синтаксису BSL (это уже сделано в предыдущих книгах серии).
- Не сборник готовых рецептов «как сделать отчёт X».
- Не путеводитель по конкретной типовой конфигурации.
- Не история платформы 1С (история — только как «награда» в C5).

## Метафора книги

Книга — это **разговор с опытным коллегой**, который сначала показывает на знакомых
бытовых вещах (фотоальбом, история операций по карте, лог событий, календарь), потом
переводит на язык платформы и только в конце — на язык предметки. Не лекция, не справочник.

## Принципы изложения (жёсткие)

1. **Граница доменов жёстко.** Платформа (как хранится/считается) и предметка (что значит) —
   не смешиваются. На уровнях 0–1 — только платформа.
2. **Упрощение = частный случай, не противоречие.** Позднее уточнение достраивает раннее,
   а не сносит. Перед добавлением детали: расширит или развернёт?
3. **Термин — награда, а не входной билет.** Порядок: *пример из жизни → мостик → ярлык*.
   Абзац не открывается незнакомым термином.
4. **Учим на родном домене ученика.** Регистр не знает, что он «бухгалтерия».
5. **Цена механизма.** Сложнее регистр → больше возможностей и дороже сопровождать.
   Выбор РН вместо РБ/РР — архитектурное решение, не «попроще».

## Что автор не будет делать никогда

- Открывать материал предметным жаргоном без бытового моста.
- Давать «откуда→куда = Кт→Дт» как мнемонику (ломается на пассивных счетах).
- Использовать образ **«сообщающиеся сосуды»** для РБ (подразумевает выравнивание уровней).
- Подавать регистр расчёта как «накопление с допсвойствами».
- Делать тяжёлое форматирование в тексте глав. Только проза.
- Менять канон метафор (см. CLAUDE.md §4) без записи в CHANGELOG.

## Аудитория (кратко)

**Читатель:** первокурсник, прочитавший предыдущие книги серии. Умеет читать BSL и метаданные,
видит структуру конфигурации, но регистры видел только вскользь.

**Читатель НЕ:** бухгалтер, расчётчик, опытный 1С-программист, аналитик с конкретной задачей,
человек без знакомства с серией.
