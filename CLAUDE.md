## 1. О проекте

- **Что**: учебная книга о регистрах «1С:Предприятие 8».
- **Аудитория**: первокурсник, уже умеющий читать BSL и метаданные по предыдущим книгам серии.
- **Цель**: довести от общего скелета до осознанного выбора регистра без перегрузки.

## 2. Два домена и граница (жёстко)

- **Платформа** — регистры, измерения, ресурсы, виртуальные таблицы, остатки/обороты/срез.
- **Предметка** — счёт, сальдо, дебет, кредит, проводка, регламент.
- На уровнях 0–1 говорим только **как** хранится и считается, а не **что** значит.
  Остаток/оборот/срез — это платформа (есть у регистра независимо от смысла). Предметка
  начинается на словах счёт/сальдо/Дт/Кт/проводка и вводится точечно, только в фазе C.

## 3. Педагогические правила (для любого текста)

1. **Упрощение = частный случай, не противоречие.** Позднее уточнение достраивает, не сносит.
   Перед добавлением детали спросить: она *расширит* или *развернёт* раннюю картинку?
2. **Термин — награда, а не входной билет.** Порядок: *пример из жизни → мостик → ярлык*.
   Никогда не открывать абзац незнакомым термином.
3. **Учим на родном домене ученика** (телефон, кошелёк, лог, расписание). Регистр не «бухгалтерия».
4. **Минимальная доза второго домена, точечно** (перед РБ/РР).
5. **Три уровня**: 0 — бытовой вход; 1 — основной курс; 2 — экспертный конспект (пункт назначения).
6. **Цена механизма.** Сложнее регистр → больше возможностей и дороже сопровождать.
   Выбор РН вместо РБ/РР — архитектурное решение, не «попроще».
7. **Низкая когнитивная нагрузка.** Одно новое за раз. Сначала образ, потом код.

## 4. Канон метафор и якорей (менять только через CHANGELOG)

- Этимология: *regerere* → книга записей. Регистратор = кто вносит; движение = запись; регистр = книга.
- Регистр = след глагола-документа.
- Сведений — «фотоальбом»: отдаёт последнюю фотографию **до заданной даты** (не «свежую добавленную»).
- Накопления — «история операций по карте»; формула: остаток — сколько накопилось к моменту,
  оборот — сколько прошло за период (оборотный ↔ «счётчик шагов»).
- Бухгалтерии — «−тут +там» (проводка) → «сеть переводов, каждый сходится». **Не** «сообщающиеся сосуды».
- Расчёта — «копилка с машиной времени» (ярлык) + для айтишника «вытесняющая многозадачность /
  календарь с правилами пересечения». Раскрытие — только в C2.
- **Два разговара** (именованный концепт, ввод в A4): пишу движения / спрашиваю виртуальную таблицу.
- **Дебет/Кредит — без направленной мнемоники.** Не «откуда→куда». «Две стороны записи, смысл
  зависит от типа счёта»; содержательно — в C1.

## 7. Формат параграфа (серия)

Зачин → содержательные разделы → итоговый абзац-сводка → контрольные вопросы → упражнения →
упражнение на «чужой форме»/чужом коде → ответы в трёх уровнях → мостик к следующему параграфу.

## 8. Практический хребет

- Лестница: читать → трассировать по образцу → копировать → писать по памяти → писать с ловушками
  → **вердикт до постройки** (проверить, нет ли готового решения).
- На каждый стоп-сигнал плана §10 — trap-задача с разбором.
- Веха на фазу — измеримый признак перехода (см. PLAN §11).

## 9. Стиль письма

- Тон человеческий, **без канцелярита** («следует отметить», «представляется целесообразным» — нельзя).
- Объяснять «зачем», а не только «что». Предложения короткие: ориентир 10–18 слов.
- Адресат — «студент» / «первокурсник» (прямое обращение уместно; отличие от edu-framework сознательное).
- Код с §B и далее: `;` после каждой команды, `КонецЕсли`/`КонецЦикла`/`КонецПроцедуры`,
  комментарии только через `//`. Сквозной 1С-пример — «Канцтовары» (бытовые домены — до конфигуратора).

## 10. Преемственность с серией

Супра-метафора «1С как иностранный язык»: существительное→справочник, прилагательное→реквизит,
глагол→документ, регистр→книга учёта. Формулировки про регистры — из ООП-цикла. Где аналогия
молчит (независимые РС, пограничные случаи) — говорить прямо, не натягивать.

## 11. Фактура — проверка перед релизом

- Имена виртуалок расчёта (`База<Имя>`, `ДанныеГрафика`, `ФактическийПериодДействия`) —
  **сверить в синтакс-помощнике** целевой версии до релизного ревью (field-vs-VT — зона прокола).
- Корреспонденция в РБ опциональна — подтверждено.
- КАМИН/7.7 — давать **сноской** с пометкой «по свидетельствам практиков, первичного подтверждения нет».
- Любое фактическое утверждение о платформе — сверять с документацией; спорное — в `docs/` «на проверку».

## 12. Нельзя

- Открывать материал предметным жаргоном без бытового моста.
- Смешивать платформу и предметку в одном объяснении.
- Давать «откуда→куда = Кт→Дт» как мнемонику.
- Тяжёлое форматирование в тексте глав. Проза.
- Пушить в `main` напрямую; удалять файлы с историей (статус `deprecated`).
- Менять канон метафор без записи в CHANGELOG.

## 13. Рабочий процесс (Claude Code)

- Claude **предлагает терминальные команды** для ФС и git, **не выполняет их сам** и не пишет
  напрямую в локальные файлы.
- Перед созданием файлов — разведпроход: резюме структуры/конвенций/конфликтов, затем план на утверждение.
- Ревью глав — через DeepSeek (`scripts/review.ps1`), результат локально (`reviews/`),
  решение — в `.review-status.json`.
- PowerShell: UTF-8 BOM, русский вывод, идемпотентность.
- Если `create_file` падает на файлах > ~250 строк — писать через `cat > file << 'EOF'`.
- Драфт с AI — отметка в PR.
